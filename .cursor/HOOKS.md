# Cursor Hooks

Hooks автоматизируют действия после событий. Конфигурация в `.cursor/hooks.json`.

**Типы хуков:**
- `afterFileEdit` - после редактирования файла (доступна `$CHANGED_FILE`)
- `subagentStop` - после завершения subagent (доступны `$SUBAGENT_TYPE`, `$STATUS`)

## AI Prompts для создания хуков

Скопируй нужный промпт, заполни параметры `[В_КВАДРАТНЫХ_СКОБКАХ]`, отправь AI.

---

### 🎨 Auto-format после изменений

```
Создай hook для автоформатирования файлов после редактирования.

Тип хука: afterFileEdit
Инструменты форматирования: [prettier/black/gofmt/rustfmt]
Расширения файлов: [.ts,.tsx,.js,.jsx]
Путь к скрипту: .cursor/hooks/format.sh

Хук должен:
- Срабатывать только на файлы с указанными расширениями
- Быть быстрым и не блокировать при ошибках
- Обновить .cursor/hooks.json
```

---

### 🔧 Lint-fix после работы worker

```
Создай hook для автоисправления lint-ошибок после завершения worker agent.

Тип хука: subagentStop
Линтер: [eslint/pylint/clippy]
Расширения файлов: [.ts,.tsx,.js,.jsx]
Путь к скрипту: .cursor/hooks/lint-fix.sh

Хук должен:
- Срабатывать только для worker агента со статусом completed
- Исправлять только staged файлы
- Запускать линтер с --fix --quiet
- Не падать при ошибках
- Обновить .cursor/hooks.json
```

---

### 🧪 Запуск тестов после изменений

```
Создай hook для автозапуска тестов после работы агента.

Тип хука: subagentStop
Тест-раннер: [jest/pytest/cargo test/go test]
Фильтр агентов: [worker,debugger] (или all)
Команда тестов: [npm test/pytest/cargo test]
Путь к скрипту: .cursor/hooks/test.sh

Хук должен:
- Запускать только тесты связанные с изменёнными файлами
- Работать быстро (например, только unit-тесты)
- Не блокировать workflow при падении тестов
- Обновить .cursor/hooks.json
```

### 🔄 Кастомный hook

```
Создай hook для [ОПИШИ_ЧТО_НУЖНО_ДЕЛАТЬ].

Тип хука: [afterFileEdit/subagentStop]
Условия срабатывания: [когда срабатывать]
Путь к скрипту: .cursor/hooks/[ИМЯ].sh

Хук должен:
- [ТРЕБОВАНИЕ_1]
- [ТРЕБОВАНИЕ_2]
- Быть быстрым и не блокировать при ошибках
- Обновить .cursor/hooks.json
```

---

## Удаление хука

```
Удали hook [ИМЯ_ХУКА] из .cursor/hooks.json и удали соответствующий скрипт.
```

---

## Tips для AI

- Хуки должны быть быстрыми (< 5 сек)
- Всегда используй `|| true` или `set +e` для non-blocking
- Проверяй переменные окружения (`$SUBAGENT_TYPE`, `$STATUS`)
- Делай скрипты executable (`chmod +x`)
- Логируй действия с emoji для читаемости
